Introduction

Form-Based Docs are documents or texts generated as a result of filling a form. Depending on your answers the text generated will vary. This app gives you the possibility to create both the form and the text.

Text input

The most simple input. Just locate the identifier right where you want to show the input of the user. The user will type in and the identifier will be replaced by the input.

Radio-a input

Same as with the Text Input type, but in this case the identifier will be replaced by the value introduced right next to the radio button.

Radio-b input

This radio works very similar to the first radio, but now the identifier will be replaced, not with the value typed in next to the radio button, but with the value typed in the window that can be opened clicking on the icon next to it.

Radio-c input (conditional)

There will be times where you will want to prompt the user with different questions, depending on his or her answers. With this button, clicking on the icon next to it, a window will be shown where you will be able to set the questions that will be shown to the user when clicking on that particular radio button. The logic is the same as before.

Indications

In case you need to clarify or you need to set a couple of instructions on how to fill an specific field, use the right side menu to create the instructions.

Change the identifier

In order to make the form more confortable for you to manage, you can change the identifier.
